Four current testing tools...

HTML Code Sniffer bookmark

by Squizlabs

http://squizlabs.github.io/HTML_CodeSniffer/


  • BOOKMARK TO TEST ACCESSIBILITY OF A SINGLE PAGE
  • Displays pointer on page to show where element is
  • Shows line of code in error and suggests solutions
  • Can turn off warnings and only show errors

Colour contrast analyser

by Paciello Group

https://www.paciellogroup.com/resources/contrastanalyser/


  • 'EYE DROP' ELEMENTS ON A PAGE FOR COLOUR CONTRAST
  • Let's you know if you passed both AA and AAA guidelines
  • You can change colours to find a combination that works
  • Word 'colour' is spelt correctly for a change!

Axe Core library

by Deque systems

https://github.com/dequelabs/axe-core


  • ACCESSIBILITY TESTING ENGINE
  • Can be integrated into your production test environment
  • Tests your compiled DOM code
  • Can write your own rules for e2e testing
  • Open source, free to download

Axe test code sample

it('should report that good HTML is good', function (done) {
  var workingDOM = document.getElementById('working');
  axe.a11yCheck(workingDOM, null, function (result) {
    expect(result.violations.length).toBe(0);
    done();
  });
});

Tenon.io

by Karl Groves

http://tenon.io/


  • COMPLETE SOFTWARE FOR ACCESSIBILITY TESTING
  • API with JSON response you can integrate into your build tools
  • Dashboard for reports and records of previous tests
  • Change levels of testing: Certainty, Viewport Size, Browser agent
  • Test each visitor to your site
